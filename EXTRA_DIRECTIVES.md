# 추가 지시 아카이브 (추격 시스템 기획서 외)

이 파일은 `game_design_documents/`의 추격 슬라이스 기획서에 **없는** 추가 지시를 기록한다.
규칙: **사용자의 매 명령마다 이 파일을 갱신**한다. 각 지시는 요지·의도·상태로 남긴다.

---

## 2026-06-04

### D1. 노선 활성 상태 색 구분 (노선도)
- **요지:** 노선도에서 **비활성 노선은 회색**, **활성화된 시점부터 각 노선 고유색**으로 표현.
- **의도:** 어떤 노선이 활성인지 시각적으로 즉시 구분.
- **상태:** ✅ 구현·검증. `Player.ActiveLinesChanged`(초기화·환승) → `MapActivationView` → `SubwayMapRenderer.ApplyActiveLineColors`가 선·역 점을 활성=고유색/비활성=회색으로 재색칠. 회색은 렌더러 인스펙터 `inactiveLineColor`로 조절. (Play: 시작 Line1만 컬러(35)·나머지 회색(266) → Line2 환승 시 컬러 78로 증가 확인)

### D2. 확대 시 역 동그라미·선 굵기 화면 비례 고정
- **요지:** 노선도 확대 시 역 동그라미 크기와 선 굵기가 배율에 비례해 같이 커져 가시성이 떨어짐. **일정 배율 이상에서는 화면상 크기/굵기가 유지**되도록(확대돼도 일정 크기).
- **의도:** 확대해도 역·선이 비대해지지 않아 가독성 유지.
- **상태:** ✅ 구현·시각검증 완료. 줌>임계 배율에서 역·마커는 균등, 선은 굵기축만 1/zoom 역보정.
- **튜닝:** 고정 시작 배율 = **`SubwayMapZoom`(PopupPanel) → Zoom Settings → `sizeLockThreshold`** (기본 1×, 인스펙터 조절). min/maxZoom 옆에 모아둠.

### D3. 역명 텍스트 항상 최상위 렌더
- **요지:** 동그라미·선이 커지면 역명 텍스트가 가려짐. **어떤 경우에도 역명 텍스트가 동그라미/선보다 위에** 렌더링.
- **의도:** 역 이름이 항상 읽힘. (D2와 연계)
- **상태:** ✅ 구현 완료(시각검증 대기). `StationView`가 역명 라벨에 `overrideSorting` 캔버스(sortingOrder 100) 부여 → 점·선·마커 위. (마커보다도 위 — 필요 시 조정 가능)

### D4. 노선 정보(범례) UI 추가
- **요지:** 서울 지하철 노선도 범례(첨부 2번째 사진)와 유사한 노선 정보 UI를 노선도 UI에 포함. **확대/축소 영향 없음(고정 크기)**. 기본 기능: **활성 노선=고유색 / 비활성=회색**으로 노선 활성 상태를 한눈에.
- **의도:** 범례 겸 활성 상태 대시보드.
- **상태:** ✅ 구현·검증. `SubwayLineLegend`가 SubwayNetworkData에서 노선별 색 스와치+이름(한글) 자동 생성, 활성=고유색/비활성=회색, `Player.ActiveLinesChanged`에 갱신. PopupPanel 좌상단 고정(줌 비영향). (Play: 12행 생성, 시작 1호선만 컬러 → Line2 환승 시 2호선 컬러 전환 확인)
- **튜닝:** 위치·행 높이·스와치 크기·패널 폭·배경/회색은 LineLegend 인스펙터에서 조절. 폰트=Korail M SDF.

### D5. SubwayMapButton 클릭 시 노선도 팝업 (버그 수정)
- **요지:** SubwayMapButton을 눌러도 노선도 UI 팝업이 안 뜸. 클릭 → 팝업 표시되도록 수정.
- **원인:** 버튼 onClick 미연결 + PopupPanel이 비활성/SubwayMapPopup이 활성(반대로) 상태.
- **상태:** ✅ 해결·검증. SubwaySceneController가 코드로 버튼 onClick→Show 연결, 시작 시 SubwayMapPopup 닫힘·PopupPanel 켜둠. (Play: 시작 닫힘→클릭 시 열림 확인)

### D7. 승강장·지상(외부) 공간 구현 → 검증 루프 완성
- **요지:** 추격 재미 검증을 위해 승강장·지상 공간을 구현(공간은 비어 있어도 됨). 환승/재탑승/반대방향탑승/외부 작품활동이 실제 동작해 **명성도·수배도·환승 시스템**을 검증 가능하게.
- **상태:** ✅ S1~S6 구현·Play 검증 완료. 한 바퀴(지하철→승강장→지상→작품활동→복귀→재탑승) 통과, 상태 유지·씬 로드/언로드·맵 토글 정상. (scene_system_spec + 추격 기획서 확정 미구현분 전부)
- **결정:** ①공간=**멀티 씬**(PlatformScene/OutsideScene 실제 로드, 영속 매니저+런타임 로케이터) ②시작=지하철(시청)+디버그로 지상 이동(§12 시작시퀀스는 나중) ③수배도 디버그=**명성 슬라이더로 통일**(수배도는 명성 구간 자동 환산).
- **모델(scene_system_spec):** 명성(float)→수배도 레벨(경계 5/25/45/75/200). 작품활동 성공 시 명성+(상30±6/중20±4/하10±2), 실패 0. 무활동 시 명성 시간감소(하한0)→수배도 자동 하강.
- **빌드 순서(docx v2, 9단계):** ①FameSystem+WantedSystem ②SceneConfig ③지상 6버튼 ④명성감소+수배도하강 ⑤체류타이머+강제도주 ⑥승강장 4버튼+역속성 ⑦환승 노선선택 서브UI ⑧씬전환 통합+복귀 즉시 검문 ⑨TrackerManager 연동. + 추격 기획서 잔여: §10 역기능(StationData featureType), §11 시간=분 명시.
- **충돌 커버:** 승강장은 오버레이→PlatformScene 컨트롤러로 이관(도착 시 SpaceManager 씬 전환), 기존 4갈래 로직 재사용. 수배도 수동 슬라이더→명성 슬라이더 대체(WantedLevel은 WantedSystem 구동). ChaseConfig·SceneConfig 공존.
- **docx 갱신:** 2026-06-04 Downloads의 수정본을 game_design_documents에 동기화(빌드순서 7→9단계 세분화).
- **보류(기획서가 [추후/F] 지정):** 돈 경제·마스크 검문연동, 가방배치, 순찰자/경찰, 승강장 랜덤이벤트, 완성도 미니게임, 혼잡도 보정.

### D8. 하차 흐름 변경 — 도착 자동 승강장 제거, 하차 버튼으로만

### D6. 역명 텍스트 가림 잔여 케이스 (D3 보강)

- **상태:** ✅ 해결·검증. StationView.Awake에서 라벨 최상위 캔버스를 보장(매 활성화 적용) → 257/257 역 적용 확인.
