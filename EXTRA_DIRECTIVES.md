# 추가 지시 아카이브 (추격 시스템 기획서 외)

이 파일은 `game_design_documents/`의 추격 슬라이스 기획서에 **없는** 추가 지시를 기록한다.
규칙: **사용자의 매 명령마다 이 파일을 갱신**한다. 각 지시는 요지·의도·상태로 남긴다.

---

## 2026-06-04

### D1. 노선 활성 상태 색 구분 (노선도)
- **요지:** 노선도에서 **비활성 노선은 회색**, **활성화된 시점부터 각 노선 고유색**으로 표현.
- **의도:** 어떤 노선이 활성인지 시각적으로 즉시 구분.
- **상태:** 착수 예정.

### D2. 확대 시 역 동그라미·선 굵기 화면 비례 고정
- **요지:** 노선도 확대 시 역 동그라미 크기와 선 굵기가 배율에 비례해 같이 커져 가시성이 떨어짐. **일정 배율 이상에서는 화면상 크기/굵기가 유지**되도록(확대돼도 일정 크기).
- **의도:** 확대해도 역·선이 비대해지지 않아 가독성 유지.
- **상태:** ✅ 구현·시각검증 완료. 줌>임계 배율에서 역·마커는 균등, 선은 굵기축만 1/zoom 역보정.
- **튜닝:** 고정 시작 배율 = **`SubwayMapZoom`(PopupPanel) → Zoom Settings → `sizeLockThreshold`** (기본 1×, 인스펙터 조절). min/maxZoom 옆에 모아둠.

### D3. 역명 텍스트 항상 최상위 렌더
- **요지:** 동그라미·선이 커지면 역명 텍스트가 가려짐. **어떤 경우에도 역명 텍스트가 동그라미/선보다 위에** 렌더링.
- **의도:** 역 이름이 항상 읽힘. (D2와 연계)
- **상태:** ✅ 구현 완료(시각검증 대기). `StationView`가 역명 라벨에 `overrideSorting` 캔버스(sortingOrder 100) 부여 → 점·선·마커 위. (마커보다도 위 — 필요 시 조정 가능)

### D4. 노선 정보(범례) UI 추가
- **요지:** 서울 지하철 노선도 범례(첨부 2번째 사진)와 유사한 노선 정보 UI를 노선도 UI에 포함. **확대/축소 영향 없음(고정 크기)**. 기본 기능: **활성 노선=고유색 / 비활성=회색**으로 노선 활성 상태를 한눈에.
- **의도:** 범례 겸 활성 상태 대시보드.
- **상태:** 착수 예정.
